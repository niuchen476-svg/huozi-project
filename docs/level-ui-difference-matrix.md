# 七关 UI 差异清单与统一目标

## 统一原则

- 玩法机制保留，每关只统一外层视觉、提示语义、触控尺寸和完成流程。
- 展馆操作以点击和触控为主，键盘只作为辅助快捷方式。
- 深色现场承载行动，浅色档案纸承载史料、复盘和 AI 表达。
- 主要按钮触控高度不低于 48px；16:9 大屏优先保证核心任务一屏可见。

## 当前差异

| 关卡 | 当前技术形态 | 主要差异 | 本轮目标 |
|---|---|---|---|
| 瑞金出发 | 共用 2.5D 历史行动 | 键盘提示突出、场景按钮不像按钮 | 全流程点击化，接入统一按钮和反馈 |
| 湘江血战 | 共用 2.5D 历史行动 | 与瑞金共用机制，但颜色和操作提示仍偏键盘 | 全流程点击化，保留战场节奏 |
| 遵义转折 | 原生多阶段推理关 | 弹窗、纸张、按钮、布局存在多套样式 | 作为统一视觉样板，规范色彩、纸张、按钮、弹窗和16:9布局 |
| 四渡赤水 | iframe 嵌入关卡 | 内部 UI 与 Level Host 视觉断层 | 下一轮为嵌入页增加同源视觉变量和消息协议 |
| 飞夺泸定桥 | 原生动作关 | 开场、选队、行动三套视觉语言 | 保留动作表现，统一任务弹窗、按钮和结果页 |
| 雪山草地 | iframe 嵌入关卡 | 内部按钮、提示和外部顶栏样式不一致 | 下一轮统一嵌入页触控按钮、提示与人物叙事卡 |
| 会宁会师 | 原生综合关 | 时间线、数字展台较完整，但控件密度不同 | 对齐统一按钮、纸张、复盘和展台完成仪式 |

## 统一组件清单

1. Level Host：关卡身份、五段式进度、返回和重来。
2. 任务开始弹窗：身份、任务问题、主操作、次操作。
3. 反馈系统：普通、成功、错误、辅助、史实确认五类。
4. 史料抽屉：统一入口、纸张卡、图片、来源与审核状态。
5. 完成复盘：行动总结、历史意义、碎片奖励。
6. AI 表达：本关角度、短输入、结果朗读和离线状态。
7. 展馆适配：48px 触控、16:9 安全区、无键盘完成全流程。
