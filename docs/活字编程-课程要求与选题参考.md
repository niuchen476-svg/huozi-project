# 活字编程 · 复活我们的图书馆 —— 课程要求与选题参考

> 来源：https://zwy.alimq.com/huozi/index.html （浙江大学人工智能学院短学期课程）
> 本文件整理给 Claude Code 使用，作为项目背景与需求说明。

## 一、课程概况

- **名称**：活字编程 · 复活我们的图书馆
- **主办**：浙江大学人工智能学院（短学期课程）
- **规模**：61 位同学 + 15 位少年（K12），约 15 组
- **时间**：7/6–7/17，每日 9:00–17:00
- **地点**：东区教学楼 1B-411（D6 上午、D10 全天在浙大图书馆）
- **平台/资源方**：造物云（提供 Vibe Coding 编程工作台与算力，零环境配置、浏览器即写即跑）、Tripo3D（赞助 3D 模型生成能力）、浙大图书馆 × CADAL（提供古籍善本、民国影像、名人手稿等独家素材与命题）
- **主题**：图书馆是背景，未来是主题。核心命题——把 CADAL 里"定死"在书页上的古籍，拆成可被 AI 调用的素材，再重新排成今天的人愿意读、愿意玩、愿意用的样子（"老字模，活过来 / Old type, live again"）。
- **产出要求**：一个能在浏览器里直接预览、核心流程能走通的真实作品（demo），不是 PPT 或截图。

## 三、评分标准

两轮制：**第①段桌评海选**（四维打分 + 人气投票，筛出入围名单）→ **第②段路演决赛**（入围队伍登台 5 分钟 Demo + 答辩，决出金银铜与全场大奖）。

| 维度 | 权重 | 说明 |
|---|---|---|
| 创意与洞察 | 30% | 选题是否新颖、是否真正抓住一个人的痛点，而不是为做而做 |
| 完成度 | 30% | 能不能真点、真跑？核心功能是否闭环，而非只有截图 |
| 文化价值 | 25% | 是否真正用好了 CADAL / 馆藏的独特资源，让文化被更多人看见、读懂、用上 |
| 现场演示 | 15% | 5 分钟内能否讲清"为谁、解决了什么、怎么用" |

**重要规则**：组是一起打仗的，但每个人都要交出一件属于自己的作品——可以是团队主作品里的一块，也可以是独立做的小工具。团队拿团队奖，个人产出也单独计入完赛与单项奖。

## 四、路演前自检清单

- [ ] 能打开、能点：作品在浏览器里能直接预览，核心流程从头到尾走得通，不靠"想象"
- [ ] 一句话讲清它是什么："我给___（人）做了一个___，帮 TA ___"，练到脱口而出
- [ ] 有一个亮点时刻：演示里安排一个"哇"的瞬间——老照片复活、文物转起来、一搜即中
- [ ] 用了真实的馆藏素材：至少接入一份真的 CADAL / 图书馆数据，而非全是占位假数据
- [ ] 备好兜底：网络/接口可能抽风，提前录一段演示视频或截图，万一现场出问题能顶上

## 五、选题方法论

### 万能公式
**资源 × 人群 × AI 能力 × 产品形态 = 一个好选题**

- **① 资源**：古籍善本 / 民国报刊 / 名人手稿 / 老照片 / 学位论文 / 馆藏书目 / 戏曲音视频 / 借阅选座数据
- **② 人群**：本科生 / 考研党 / 留学生 / 研究者 / 中小学生 / 银发读者 / 访校游客
- **③ AI 能力**：OCR 识别 / 断句翻译 / RAG 问答 / 知识图谱 / 图像修复上色 / 推荐 / 语音 / 生成插画
- **④ 产品形态**：网页应用 / 聊天机器人 / 数据可视化 / 小游戏 / 沉浸展览 / 浏览器插件 / 小程序

话术提醒：别想"做个很厉害的东西"，先想"替谁，解决他翻这批老资料时的哪一个具体麻烦"。范围越小越好做，越具体越打动评委。

**头号风险提醒**：无论哪条赛道，数据可得性决定生死。务必尽早确认能拿到 CADAL / 图书馆方提供的样例数据或 API（哪怕几百册古籍图片、一份书目 CSV），否则第一天就会卡在"拿不到数据"上。

### 六大赛道

| 赛道 | 方向 | 示例点子 | 类型 |
|---|---|---|---|
| TRACK 01 古籍·遗产活化 | 让沉睡的善本"开口说话" | 古籍 OCR+自动断句+白话翻译注释；"与古人对话"AI 分身；古籍知识图谱（人物-地点-事件）；善本 3D 虚拟书架/沉浸式翻阅 | 偏展示·人文感强 |
| TRACK 02 阅读·发现推荐 | 把"找书"变成"懂你" | 自然语言语义搜书；个性化荐书/主题书单盲盒；"一本书的版本旅行"可视化；跨馆藏相似主题串联 | 偏落地·当天可用 |
| TRACK 03 智慧·图书馆服务 | 解决每天都在发生的真实痛点 | 图书馆 RAG 咨询机器人；找书室内导航（按索书号定位书架）；学习搭子匹配+选座热力图；文献传递/荐购智能助手 | 偏落地·评委共鸣 |
| TRACK 04 人文·数据可视化 | 让数据讲出一个故事 | 古籍刊刻地理地图；某主题出版时间轴/词频流变；名人交游网络/师承关系图；馆藏结构"全景体检"仪表盘 | 偏展示·美感取胜 |
| TRACK 05 教育·游戏化 | 把学习包装成玩 | 古籍知识闯关问答/卡牌；校园数字寻宝 City Walk；面向中小学的古籍互动课件；"修复一页古籍"体验小游戏 | 偏展示·现场出彩 |
| TRACK 06 文创·传播出圈 | 让 CADAL 被更多人看见 | 古籍纹样→生成式文创素材库；资源自动生成科普短视频脚本+配图；"每日一页古籍"社媒内容机器人；AI 策展：自动生成线上主题展 | 偏落地·传播价值 |

赛道作用是分流（避免全班扎堆做"古籍问答机器人"），建议报名时每队选 1 条主赛道。

## 六、产品定义（Define 工作坊）

在写第一行代码之前，先把要做的事说成一句话。五个必答问题 + 一刀 MVP：

1. **WHO** —— 给谁做？
2. **PAIN** —— 解决什么痛点？
3. **SCENE** —— 什么场景下用？
4. **VALUE / EDGE** —— 凭什么是你（独特性/价值）？
5. **MVP** —— 最小可行版本切一刀，先做出能跑的核心

## 七、可用工具

- **灵感骰子**：想不出题时摇一摇，随机给出"资源×人群×能力×形态"组合，破冰用
- **配方器**：选「一份知识资源 × 一种复活方法 × 一种交互形态」，自动拼出可着手的作品配方
- **产品定义画布**：九宫格填 谁/痛点/场景/价值/独特性/MVP/不做什么，自动存本机、可复制下载
- **作品矩阵**：方法×形态矩阵，看全班选题分布，找"高价值待填区"
- **AI 评一评我的定义**：填完定义画布后，AI 按"五问+MVP+红线"打分、指出问题、给出改法
- **AI 评一评我的选题**：写下一句话选题，AI 按「真实需求/资源结合/AI 用得巧/两周可行性与数据可得/演示哇点」逐项打分，重点提醒数据可得性

建议使用顺序：摇骰子破冰 → 配方器拼方向 → 定义画布收敛成一句话 → AI 评一评、按建议改一版 → 作品矩阵确认全局位置，然后开做。

## 九、常见问题（FAQ 摘要）

- **不会写代码能参加吗？** 能。这正是 Vibe Coding 的意义——用自然语言描述想要什么，AI 生成可运行代码。更重要的能力是讲清需求、把问题想透。
- 一个组几个人？怎么分工？（详见站内 FAQ）
- 组队了是不是抱大腿就行？（不是，每人需交出个人可辨识产出）
- 有中学生（K12）一起参加？（是，约 15 位少年参与，有"最受小朋友欢迎"单项奖）
- CADAL 数据从哪里拿？（见上文 CADAL 素材库）
- Tripo3D 是用来干嘛的？（一句话/一张图生成可用三维模型，用于文物复原、文创手办、沉浸场景）
