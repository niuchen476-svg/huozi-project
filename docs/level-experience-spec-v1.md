# 统一关卡体验规范 v1

## 1. 适用范围

本规范用于七个关卡的公共体验层。参考《遵义转折》样稿的五段式阶段、顶部状态、史料档案入口、弹窗层级和完成反馈，但不复制遵义关卡专属的会议室、纸张和深绿色视觉。

目标体验时长为 1～3 分钟，玩家可以从路线图任选关卡。

## 2. 统一与差异的边界

### 必须统一

- 页面安全区、顶部工具区和右上角“本关史料”入口；
- “进入历史—身处历史—遇见文献—作出表达—完成本关”的阶段语言；
- 主按钮、次按钮、关闭按钮、返回和重来位置；
- 任务提示、正确反馈、错误反馈、辅助提示和完成弹窗；
- 弹窗遮罩、关闭方式、滚动方式、键盘操作与焦点管理；
- AI 表达面板的输入、生成、朗读、失败回退和完成动作；
- 碎片获得、查看和返回路线图的仪式顺序。

### 允许关卡差异化

- 核心玩法和场景构图；
- 背景图、历史道具、局部纹理和受控的关卡强调色；
- 本关问题、提示语言、史料内容与表达任务；
- AI 结果所模拟的历史文书类型；
- 碎片模型、名称和历史含义。

原则：公共外壳稳定，历史矛盾和核心操作不同。不要让玩家每进入一关都重新学习导航和弹窗。

## 3. 五段式体验

1. 进入历史：显示时间、地点、本关问题和一个明确的开始动作。
2. 身处历史：运行本关独有玩法；首次操作前给一句短提示。
3. 遇见文献：史料直接参与判断；右上角档案入口全程可打开，但不强制点击。
4. 作出表达：玩家基于选择、史料和一句短输入形成自己的表达。
5. 完成本关：简短复盘、获得碎片、返回路线图；会宁进入数字展台收束。

阶段条表示当前所处位置，不应要求玩家为了完成阶段而阅读全部史料。

## 4. 史料抽屉

### 统一行为

- 固定在右上角，按钮文字统一为“本关史料”，同时显示数量；
- 点击后从右侧打开，背景玩法暂停，关闭后恢复；
- 支持遮罩点击、关闭按钮和 `Esc` 关闭；
- 内容区域独立上下滚动，不带动背后的关卡；
- 没有内容时仍保留入口并显示待补充状态；
- 一关建议收录 3～8 份，最多支持 12 份常规展示；其中 1～3 份真正参与玩法；
- 史料图片采用缩略展示，原文、白话说明和出处在同一条目内展开。

### 单条史料字段

```json
{
  "id": "source-zunyi-01",
  "title": "史料名称",
  "type": "会议记录 / 地图 / 照片 / 回忆录",
  "date": "形成时间",
  "creator": "作者或形成机构",
  "image": "assets/levels/zunyi-turn/sources/source-01.jpg",
  "imageAlt": "图片替代说明",
  "imageCaption": "图片说明",
  "summary": "一句话摘要",
  "originalExcerpt": "必要的短段原文",
  "plainExplanation": "面向儿童的白话说明",
  "sourceName": "馆藏、出版物或网页名称",
  "sourceUrl": "https://...",
  "rightsStatus": "pending",
  "activeInGameplay": true,
  "visibleInSourceDrawer": true,
  "availableForAiExpression": true
}
```

史实审核由项目负责人完成。未审核材料不得标记为正式上线内容。

## 5. AI 表达差异化

七关共用 Level Host、MiMo 接口、输入组件、生成状态、朗读按钮和离线模板。差异不通过复制七套页面实现，而由每关 `experience.json` 中的表达配置决定。

| 关卡 | 玩家表达任务 | AI 结果类型 | 结果卡视觉隐喻 |
|---|---|---|---|
| 瑞金出发 | 我为什么出发、最想带走什么 | 出发札记 | 行军手记 |
| 湘江血战 | 我从路线与伤亡中看到了什么 | 战地记录 | 战地记录纸 |
| 遵义转折 | 哪些材料改变了对方向的判断 | 会议摘要 | 会议记录页 |
| 四渡赤水 | 为什么路线变化不是犹豫 | 路线复盘 | 地图批注 |
| 飞夺泸定桥 | 我会如何报告这次行动 | 行动电报 | 电报稿 |
| 雪山草地 | 一个具体的人和选择留下了什么 | 行军记忆 | 回忆卡片 |
| 会宁会师 | 三块碎片如何组成我的长征理解 | 展台讲解 | 展签，并继续生成纪念画 |

### 必须保持一致

- 输入不超过一到两句话，适合展馆短时操作；
- 先显示玩家选择的史料或碎片，再让玩家输入；
- AI 内容必须和历史原文明确区分；
- 在线生成成功时不显示“离线模板”；失败时自动使用本关模板；
- 生成结果都可以通过统一的小喇叭朗读；
- 结果页必须提供“重来”和“完成/返回”动作。

### 差异化配置项

- `prompt`：本关给玩家的问题；
- `outputType`：出发札记、会议摘要、电报等；
- `outputLabel`：结果卡标题；
- `sourceSelectionLimit`：可引用史料数量；
- `maxCharacters`：玩家输入长度；
- `fallbackTemplates`：本关独立离线结果；
- 结果卡的 `documentVariant`：只改变纸张/文书外观，不改变交互布局。

## 6. 暂缓项目

- 音频本期先不制作，但保留现有数据协议，不删除字段；
- 3D 碎片等待正式模型和说明后统一接入，不用临时素材替代；
- 史料正文等待审核内容到位后逐关填写。

## 7. 验收清单

- 七关右上角均能找到同一位置、同一名称的史料入口；
- 空史料、长列表、图片加载失败时均不影响通关；
- 打开史料时玩法暂停，关闭后恢复；
- 窄屏和展馆大屏均可完整关闭弹窗；
- 五段式阶段名称、按钮用语和反馈用语保持一致；
- 每关保留独特玩法，但不新增私有的公共弹窗和导航逻辑；
- 单关正常体验可控制在 1～3 分钟。
